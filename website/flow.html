System Architecture

How
It Works

Five steps from user action to forensic evidence — everything happens in milliseconds, invisibly, before any threat reaches you.

01
🧩
Browser Extension Monitors
LAYER 1 — USER LAYER

The Chrome extension runs silently in the background using Manifest V3. It watches three things: every URL you navigate to, every file you download, and every email you open in Gmail or Outlook.

02
📡
Data Sent to Backend
LAYER 2 — COMMUNICATION LAYER

The extension sends the captured data as a JSON POST request to the Flask backend running locally. Each type of threat has its own dedicated API endpoint.

API ENDPOINTS
POST /api/analyze-url ← URL analysis
POST /api/analyze-file ← File + sandbox
POST /api/analyze-email ← AI phishing scan
GET /api/incidents/stats ← Dashboard stats
03
🧠
AI + Rule Analysis
LAYER 3 — INTELLIGENCE LAYER

The backend runs two types of analysis in parallel. Rule-based checks are fast and deterministic. The ML model adds deep pattern understanding that catches social engineering and novel attacks.

05

Forensic Evidence Logged
LAYER 5 — FORENSICS LAYER

Every medium and high risk event is stored in the SQLite database as forensic evidence — with the threat type, value, risk level, reason, numeric score, action taken, and timestamp. The dashboard shows everything.
