Project Background

About
the Project

BrowserShield is a mini project built to solve a real problem — people get hacked not because security systems fail, but because users trust what they see.

The Problem
Security Reacts Too Late

Most security tools — antiviruses, firewalls, email filters — react after a threat has been encountered. The user has already clicked the link, already downloaded the file, already entered their password.

Phishing works because the attacker exploits human trust, not technical vulnerabilities. A perfectly designed fake PayPal login page looks real. An urgent-sounding email creates panic that bypasses rational thinking.

No amount of post-incident scanning fixes this.

The Solution
Intercept Before Interaction

BrowserShield moves the security checkpoint to before the user interacts with a threat. Every URL is analyzed the moment you navigate. Every download is scanned before execution. Every email is checked when you open it.

The system combines rule-based detection for known patterns and machine learning for new ones — producing a risk decision in milliseconds, invisibly, without interrupting normal browsing.

When something is dangerous, the user is warned or the action is blocked.

Research Question
"Can a lightweight browser-integrated hybrid detection system using rule-based analysis and machine learning provide effective real-time protection against phishing emails, malicious URLs, and dangerous downloads — with acceptable false positive rates?"
This project attempts to answer this question by building the system, measuring its detection performance, and evaluating its real-world usability through a working Chrome extension connected to a Flask backend with a trained ML model.

Project Scope

What's In &
What's Next

What was built for this project and what future work would look like.

✅ Implemented
  • Flask REST API with modular Blueprint architecture
  • URL analysis with 11 detection rules
  • Email phishing detection (ML + rule hybrid)
  • File inspection (30+ extensions, magic bytes)
  • Static sandbox analysis (7-step, no execution)
  • SQLite forensics database with full logging
  • Real-time forensics dashboard
  • Chrome extension (Manifest V3)
  • Gmail and Outlook email scanning
  • Allow/Block download decision popup
  • ML model trained on SpamAssassin dataset
  • Tested in VMware virtual environment
🔮 Future Work
  • VirusTotal API integration for real-time hash lookup
  • Dynamic sandbox using isolated VM execution
  • BERT or transformer-based email classification
  • WebSocket real-time dashboard updates
  • Firefox browser extension support
  • User behavior analytics and anomaly detection
  • Threat intelligence feed integration
  • API key authentication for production security
  • Mobile companion app for alerts
  • Multi-user dashboard with role-based access
